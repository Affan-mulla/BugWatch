# 📋 Automated Pull Request Review Agent – Frontend

## 📌 Overview

The **Automated Pull Request Review Agent (Frontend)** is the user-facing interface for a GitHub-integrated static analysis system.

It allows developers to:

* Authenticate with GitHub
* Install and manage the GitHub App
* View Pull Requests and review results
* See structured AI explanations
* Compare PR code changes (diff viewer)
* Track review history and metrics

The frontend does **not** perform analysis.
It visualizes results generated by the backend rule engine + AI explanation layer.

---

# 🎯 Objectives

* Provide a clean UI for interacting with the review system
* Display structured rule-based findings
* Show PR code diffs alongside AI explanations
* Enable repository-level configuration
* Present review history and metrics clearly
* Maintain transparency (rule → issue → explanation → fix)

---

# 🏗 Core Frontend Features

## 1️⃣ Authentication & GitHub Integration

* Sign in with GitHub (OAuth)
* Display user avatar + username
* Detect GitHub App installation status
* Provide “Install GitHub App” CTA if not installed
* Repository access validation

---

## 2️⃣ Repository Management

User can:

* View repositories they have access to
* See installation status of GitHub App
* Enable/disable automated review per repository
* View repository-specific review statistics

---

## 3️⃣ Pull Request Dashboard

For each connected repository:

* List recent Pull Requests
* Show:

  * PR title
  * Author
  * Status (open/closed/merged)
  * Number of issues found
  * Severity breakdown (High / Medium / Low)
* Click PR → Open detailed review page

---

## 4️⃣ PR Review Details Page

For a selected PR:

Display:

* PR metadata (author, branch, commit count)
* Files changed
* Total issues detected
* Review timestamp

Each issue must show:

* File name
* Line number
* Rule violated
* Severity level
* Deterministic rule explanation
* AI-generated explanation
* Suggested fix
* Code snippet context

Issues should be grouped by:

* Security
* Logic
* Pattern
* Performance

---

## 5️⃣ Code Diff Viewer (Critical Feature)

The frontend must include a **side-by-side or inline diff viewer**.

Features:

* Display only changed files
* Highlight additions and deletions
* Show line numbers
* Scroll synchronization
* Click on a line → highlight related issue
* Overlay issue markers (like GitHub)

Example behavior:

User opens PR →
Sees diff →
Red badge appears near line 24 →
Click → shows rule violation + AI explanation panel

This is what makes it feel professional.

Recommended capabilities:

* Syntax highlighting
* Large file handling
* Collapsible unchanged sections
* File tree navigation

---

## 6️⃣ AI Review Presentation Layer

Each detected issue should clearly separate:

Rule-based detection (deterministic)
AI explanation (natural language)

UI should show:

Rule: no-eval
Severity: High
Why this matters: Security risk due to dynamic code execution
Suggested fix: Replace eval with safe parsing

Transparency is mandatory.

AI must not appear as decision-maker.
It explains — it does not judge.

---

## 7️⃣ Review History & Metrics Dashboard

Display:

* Total PRs analyzed
* Total issues detected
* Most frequent rule violations
* Security vs Logic ratio
* Repository-level trends
* Time-based issue graphs

Purpose:

* Track technical debt
* Identify recurring mistakes
* Show system impact

---

## 8️⃣ Settings & Configuration

Per repository:

* Enable/disable rule categories
* Toggle AI explanation layer
* Configure severity thresholds
* Delete stored review history

---

## 9️⃣ Notification System (Optional but Recommended)

* Review completed alerts
* High severity issue notifications
* Repository configuration updates

---

# 🧠 Design Principles

Every issue must show:

* Rule violated

* Severity

# 🔄 Frontend Workflow

1. User logs in with GitHub
2. App verifies GitHub App installation
3. User selects repository
4. Backend receives webhook when PR is created/updated
5. Backend performs:

   * AST parsing
   * Rule analysis
   * AI explanation generation
6. Frontend fetches review results
7. PR page displays:

   * Code diff
   * Inline issue markers
   * AI explanation panel
8. Review history stored and visualized
